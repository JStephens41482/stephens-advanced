## CRITICAL FINDING: App does not load in automated browser

**16 of 16 failures share the same root cause:** The loading overlay (`#app-loading`) never disappears in the Playwright browser. The app gets stuck at "Loading..." and never reaches the interactive state.

This means **Supabase `load()` is failing or hanging** when accessed from Playwright's Chromium browser. The PIN screen may also be blocking — Playwright doesn't know the PIN.

### Root Cause Analysis:
1. The app requires PIN authentication (`checkPin()`) before `startApp()` runs
2. Playwright can't unlock the PIN because it doesn't know the code
3. Even if PIN is bypassed, the loading overlay blocks all interaction

### Impact:
- ALL app tests fail (auth, customer, jobs, money, techs, route, debounce, performance)
- Portal tests also fail (portal page may redirect or have different structure than expected)
